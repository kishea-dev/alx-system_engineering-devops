# Postmortem


## Sample:

```
 Post-Mortem: [Incident Name]

 Background:

 [Brief description of the incident, including date/time, impact, and affected systems/services.]

 Timeline:

 [Chronological list of events, from incident detection to resolution.]

 Root Cause Analysis:

 [Description of what went wrong, what caused the incident, and underlying problems.]

 Remediation:

 [Steps taken to resolve the incident and prevent it from happening again.]

 Lessons Learned:

 [Key findings and changes to processes/procedures to prevent similar incidents in the future.]

 Conclusion:

 [Summary of the post-mortem, importance of conducting regular post-mortems, and thanks to relevant stakeholders.]

```
